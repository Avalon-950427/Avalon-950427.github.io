信息 = 位 + 上下文

源文件(源程序)是一个由 0 和 1 组成的位(bit)序列,8 个位组成一个字节。

这个例子说明了一个基本思想:系统中所有的信息——包括磁盘文件、内存中的程序和数据以及网络上传送的数据，都是遗传 bit 表示的。区分不同数据对象只能通过读数据的上下文（一个同样的字节序列可能表示证书、浮点数或机器指令）。

程序被其他程序翻译成不同的格式

上面这段程序是一段能被人读懂的 c 语言程序，但是要想在系统上运行，每条 c 语句必须被其他程序转化为低级机器语言指令，然后指令被称为可执行目标程序的格式打包好，并以二进制的形式存储。

  1. 预处理阶段：根据#字符开头的命令，修改原始程序。如#include <stdio.h>告诉预处理器读取系统头文件并插入程序文本中，结果得到一个扩展名为.i 的另一个 c 程序
  2. 编译阶段：编译器将 hello.i 翻译成包含汇编语言程序的 hello.s。
  3. 汇编阶段：汇编器在这个时候将 hello.s 翻译成机器语言，并打包成可重定位目标程序的格式，并将结果保存在一个二进制文件 hello.o 中
  4. 链接阶段：因为 hello.c 调用了 printf 函数，每个 c 编译器都提供这个函数，而 printf 函数单独存在于 printf.o 的预编译好的文件中，此时链接器就负责处理这种合并，链接结果得到一个可执行目标文件，这个文件可以加载到内存中被系统执行。

处理器读取并解释存储在内存中的指令

系统的硬件组成
  1. 总线：是一组贯穿整个系统的电子管道，它携带信息系并负责在各个部件间传递。通常总线被设计成传送定长字节块，也就是字。
  2. I/O 设备：是系统与外部世界连接的通道。每个 I/O 设备都通过一个控制器（I/O 设备本身）或适配器（插在主板上的卡）与 I/O 总线相连。
  3. 主存：是一个在处理器处理程序时用来存放程序和程序处理的数据的临时存储设备。在物理上，主存是一组动态随机存取存储器（DRAM）组成；逻辑上，存储器是一个线性的字节数组，每个字节都有对应的地址（数组索引）。
  4. 处理器：解释和执行存储在主存中指令的引擎，它的核心是一个大小为一个字的存储设备（或寄存器），称为程序计数器（PC），PC 任何时候都指向主存中的某条指令
执行 hello 程序

当通过 shell 输入./hello 的时候，在键盘上输入的字符，shell 会逐一读入寄存器，再把它放到内存中，敲下回车 shell 知道这个命令已经结束，系统判定这不是一个内置的 shell 命令，shell 会假设这个命令是可执行文件的名字，并去加载这个文件
